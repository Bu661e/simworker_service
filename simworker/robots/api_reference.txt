当前 robot 可用 API 如下：

1. pick_and_place(
       pick_position: list[float],
       place_position: list[float],
       rotation: list[float] | None = None,
       grasp_offset: list[float] | None = None,
   ) -> None

   参数说明：

   - pick_position
     - 含义：抓取阶段使用的“物体中心点”世界坐标。
     - 格式：`[x, y, z]`
     - 单位：米
     - 坐标系：world
     - 典型来源：`objects[i]["pose"]["position_xyz_m"]`
     - 注意：这里表达的是物体中心点，不一定就是最终夹爪接触点。如果物体很高、很长，或者希望抓偏一点的位置，应配合 `grasp_offset` 使用。

   - place_position
     - 含义：放置完成后，物体中心点希望到达的目标世界坐标。
     - 格式：`[x, y, z]`
     - 单位：米
     - 坐标系：world
     - 注意：这里同样表达的是物体中心点，而不是桌面接触点。如果要做堆叠，需要调用方自己先把目标中心点高度算出来。

   - rotation
     - 含义：夹爪执行抓取/放置时使用的目标朝向。
     - 格式：`[w, x, y, z]`
     - 坐标系：world
     - 默认值：`None`
     - 当为 `None` 时：使用内部默认抓取姿态。
     - 适用场景：当物体具有明显方向性时应显式传入，例如长条物体、需要沿某个方向对齐抓取的物体、或者需要控制放置朝向的任务。

   - grasp_offset
     - 含义：夹爪目标点相对物体中心点的固定偏移。
     - 格式：`[dx, dy, dz]`
     - 单位：米
     - 坐标系：world
     - 默认值：`None`
     - 当为 `None` 时：等价于 `[0.0, 0.0, 0.0]`
     - 执行语义：这个偏移会同时作用在抓取阶段和放置阶段。
     - 适用场景：
       - 物体中心点不适合作为真实抓取点时，例如高物体、长物体、希望抓取物体上表面附近的位置
       - 物体中心点的 `z` 明显低于希望夹取的高度时，可以给一个正的 `dz`
       - 如果希望放置后的实际接触点也相对中心点有同样偏移，可以直接复用这个参数

约定：
- 如果需要堆叠，请自行计算目标中心点的 `z` 高度
- 如果物体尺寸较大，请不要直接把“桌面接触点”或“物体顶部点”当成 `pick_position` / `place_position`；这两个参数都应优先理解为“物体中心点”
- 当中心点不适合直接抓取时，优先通过 `grasp_offset` 调整真实抓取/放置点
- 当物体有明显方向要求时，优先通过 `rotation` 显式指定夹爪朝向
